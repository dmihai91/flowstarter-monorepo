Flowstarter is a comprehensive business website building platform that combines beautiful professional templates, **AI-powered code generation** using multi-agent orchestration (AG2 framework), and powerful analytics to help entrepreneurs and businesses establish their online presence quickly and effectively.

## What is Flowstarter?

Flowstarter transforms the way businesses create their online presence by providing:

- 🎨 **Professional Templates** - 3 premium templates (Personal Brand Pro, Local Business Pro, SaaS Product Pro) with modern design systems
- 🤖 **AI Code Generation** - Multi-agent system using AG2 (formerly AutoGen) framework for intelligent code generation
- 🔄 **Real-time Streaming** - Live progress updates via Server-Sent Events (SSE) during website generation
- 📊 **Business Analytics** - Track visitors, leads, conversion rates, and business performance in real-time
- ⚡ **Instant Deployment** - One-click deployment to custom domains with enterprise-grade hosting
- 💼 **Lead Management** - Built-in CRM to capture and manage customer inquiries
- 📱 **Mobile-First Design** - All templates are fully responsive and optimized for mobile devices

## 🚀 Getting Started

### Integrations OAuth (MVP)

Set these in .env.local:

- GOOGLE_OAUTH_CLIENT_ID=
- GOOGLE_OAUTH_CLIENT_SECRET=
- NEXT_PUBLIC_BASE_URL=http://localhost:3000

Flow

- Start at /dashboard/integrations → “Connect via Wizard”
- Google OAuth starts at /api/integrations/google-analytics/oauth/start (PKCE)
- Callback stores tokens in HttpOnly cookie (MVP) and redirects back to the wizard
- Resources are fetched from Google Analytics Admin API v1beta

Caveats

- This is an MVP; move tokens to DB storage and implement refresh logic for production.

### For Developers

#### Prerequisites

- Node.js 18+ and yarn/npm
- Supabase account
- Clerk account (for authentication)
- Vercel account (for deployment)

#### Installation

```bash
# Clone the repository
git clone https://github.com/flowstarter/flowstarter.git
cd flowstarter

# Install dependencies (uses pnpm)
pnpm install

# Set up environment variables
cp .env.example .env.local

# Configure Supabase
pnpm db:link
pnpm db:pull
pnpm gen:types

# Start development server with Turbopack
pnpm dev
```

#### Development Scripts

```bash
# Start Next.js dev server with Turbopack
pnpm dev

# Start AI Coding Agent service (Python/FastAPI)
pnpm dev:coding-agent

# Restart AI Coding Agent service
pnpm dev:restart-coding-agent

# Run both Next.js and Coding Agent together
pnpm dev:all

# Run unit tests
pnpm test

# Run tests in watch mode
pnpm test:watch

# Run E2E tests with Playwright
pnpm test:e2e

# Lint and format code
pnpm lint
pnpm lint:prettier

# Type check
pnpm tsc

# Generate Supabase types
pnpm gen:types

# Generate template thumbnails
pnpm gen:thumbs

# Generate template previews
pnpm gen:previews

# Build for production
pnpm build

# Database commands
pnpm db:start    # Start local Supabase
pnpm db:pull     # Pull remote schema
pnpm db:push     # Push migrations
pnpm db:reset    # Reset database
```

#### Environment Variables

See [`env.example`](env.example) for a complete list of all required and optional environment variables. Copy this file to `.env.local` and fill in your values:
