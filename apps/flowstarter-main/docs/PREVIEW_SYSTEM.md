## How Preview Should Work

When a user clicks "Preview" on a template, they should see the template with THEIR project data injected.

### Option 1: IFrame Preview (Recommended)
Serve templates through an iframe that injects project data:

1. **Create Preview Route**: `/src/app/templates/[templateId]/page.tsx`
   ```typescript
   export default function TemplatePreview({ 
     params, 
     searchParams 
   }: { 
     params: { templateId: string }, 
     searchParams: { [key: string]: string } 
   }) {
     // Get project data from searchParams or session
     const projectData = {
       PROJECT_NAME: searchParams.name || 'Your Business Name',
       PROJECT_DESCRIPTION: searchParams.description || 'Your business description',
       TARGET_USERS: searchParams.targetUsers || 'your target customers',
       BUSINESS_GOALS: searchParams.goals || 'your business goals',
       PROJECT_NAME_SLUG: searchParams.slug || 'your-business',
     };
     
     // Load template and inject data
     // Return rendered template
   }
   ```

2. **Template Data Injection**:
   - Read template HTML/JSX from `/templates/{templateId}/app/page.tsx`
   - Replace all `{{VARIABLE}}` placeholders with actual project data
   - Render the result

3. **Preview URL Example**:
   ```
   /templates/personal-brand-pro?name=Sarah+Anderson+Consulting&description=Strategic+business+consulting...
   ```

### Option 2: LocalTemplateService Integration
Your existing `LocalTemplateService` should:

1. **Read Template Files**:
   ```typescript
   async readTemplate(templateId: string) {
     const templatePath = path.join(process.cwd(), 'templates', templateId);
     // Read template files
   }
   ```

2. **Replace Variables**:
   ```typescript
   replaceVariables(templateContent: string, projectData: ProjectData) {
     return templateContent
       .replace(/\{\{PROJECT_NAME\}\}/g, projectData.name)
       .replace(/\{\{PROJECT_DESCRIPTION\}\}/g, projectData.description)
       .replace(/\{\{TARGET_USERS\}\}/g, projectData.targetUsers)
       .replace(/\{\{BUSINESS_GOALS\}\}/g, projectData.businessGoals)
       .replace(/\{\{PROJECT_NAME_SLUG\}\}/g, projectData.slug);
   }
   ```

3. **Serve Preview**:
   ```typescript
   async servePreview(templateId: string, projectData: ProjectData) {
     const template = await this.readTemplate(templateId);
     const processedTemplate = this.replaceVariables(template, projectData);
     return processedTemplate;
   }
   ```

## Variable Placeholders in Templates

All templates use these placeholders:
- `{{PROJECT_NAME}}` - Business/project name
- `{{PROJECT_DESCRIPTION}}` - Full description
- `{{TARGET_USERS}}` - Target audience
- `{{BUSINESS_GOALS}}` - Business objectives
- `{{PROJECT_NAME_SLUG}}` - URL-friendly name

## Quick Fix for Testing

To quickly test previews with sample data:

1. Start the Flowstarter dev server: `pnpm dev`
2. The templates are Next.js apps, so they need their own dev server
3. For quick preview, you could:
   - Serve templates directly: `cd templates/personal-brand-pro && npm install && npm run dev`
   - This runs the template on its own port (e.g., 3001)
   - Point preview iframe to `http://localhost:3001`

## Long-term Solution

Integrate templates into your main Next.js app as dynamic routes that:
1. Read template from `/templates/` directory
2. Process and inject project data
3. Render template within your app context
4. No need for separate template servers

This way previews show user's actual data and gradients work perfectly!
